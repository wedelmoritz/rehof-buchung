Was kannst du hier tun? (ein-/ausklappen)

Zwei Rollen: Backend (hier) = Admin/Superuser – Stammdaten, Buchungen ändern, Losung starten. Verwaltung = Mitglieder der Gruppe „Verwaltung“ – nur das operative Dashboard unter /verwaltung/ (Buchungen/ Losung ansehen + Hofladen-Katalog pflegen), kein Backend. Eine Person bekommt die Verwaltungs-Rolle, indem du sie unter Benutzer der Gruppe „Verwaltung“ hinzufügst.

Im Backend pflegst du die Stammdaten (Personen, Anteile, Quartiere, Regeln, Perioden, Hofladen). Das operative Tagesgeschäft – Reinigungsliste, anstehende Buchungen, Rechnungen mahnen, Kontoabgleich – läuft im Verwaltungs-Dashboard.

Die wichtigste Beziehung: Benutzer → Mitglied → Anteil

Benutzer — 1:1 → Mitglied — über Anteil-Zuordnung → Mitglieds-Anteil

  • Benutzer = das Login-Konto (E-Mail/Benutzername + Passwort). Wer sich einloggt, ist ein Benutzer. Staff-Konten sehen die Verwaltung.
  • Mitglied = das Buchungs-Profil genau eines Benutzers (Anzeigename, Rechnungsdaten/IBAN, Karma). Ohne Mitglieds-Profil kann niemand buchen – bis es zugeordnet ist, landet die Person auf der Warteseite. Reine Verwaltungs-Konten brauchen kein Profil.
  • Mitglieds-Anteil = eine Vielleben-eG-Nummer mit Jahres-Tagebudget (Standard 50 Tage, davon 25 über die Wunschliste). Hier kommen die Tage her.
  • Anteil-Zuordnung verbindet beide: ein Mitglied hängt mit einem festen Tage-Anteil an einem oder mehreren Anteilen. Voll-Mitglied = ein Nutzer hält den ganzen Anteil; Teil-Mitglied / Tandem = mehrere Nutzer teilen sich einen Anteil. Hängt ein Nutzer an mehreren Anteilen, ist sein Budget die Summe.

💡 Jemanden freischalten: Unter Benutzer das Konto öffnen → das Mitglieds-Profil (gleich im selben Formular) ausfüllen → dann unter Mitglieds-Anteile den passenden Anteil wählen und die Person mit ihrem Tage-Anteil hinzufügen. Erst dann kann sie buchen.

Die Bereiche im Überblick

  • Benutzer – Login und Mitglieds-Profil in einem Formular (Freischaltung).
  • Mitglieds-Anteile – eG-Nummern mit Tagebudget; Zuordnung der Nutzer (Voll/Teil/Tandem).
  • Quartiere & Äquivalenzklassen – die buchbaren Unterkünfte; gleichwertige Quartiere bilden eine Klasse (die Losung darf innerhalb ausweichen).
  • Buchungsregeln – globale Mindestnächte + Saison-Regeln (Mindestnächte/Parallel-Limit/Deckel) und Schulferien.
  • Buchungsperioden – pro Jahr eine; Termine (Wünsche, Losung, buchbar ab/bis) und die Aktion „Losung durchführen“.
  • Losdurchläufe – eine gelaufene Losung bestätigen (veröffentlichen) oder zurücknehmen.
  • Betriebs-Einstellungen – Empfänger der Verwaltungs-Mails (Verwaltung/Reinigungsteam), Monats-Mail-Tag.
  • Hofladen – Produkte/Gruppen, Einkäufe, Rechnungen (mahnen/exportieren), Zahlungseingänge (Kontoabgleich).
  • Rechtliche & Zahlungs-Einstellungen – übergreifend für Hofladen UND externe Gäste: Genossenschaftsdaten (Name, Anschrift, Steuernummer/USt-IdNr., IBAN/BIC, Vorstand, Kontakt), Umsatzsteuer (Kleinunternehmer §19 an/aus), Online-Bezahlung (Mollie: an/aus + API-Key; ohne Key ein Test-Modus) sowie Impressum/Datenschutz/AGB.
